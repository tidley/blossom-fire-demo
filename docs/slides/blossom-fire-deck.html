Blossom Fire

Sovereign-first live media

Nostr control plane + Blossom blob transport.

Practical decentralized streaming for unstable networks and user-owned trust boundaries.

The core problem

  • Most “decentralized” stacks still depend on centralized control points.
  • Identity, access policy, and transport are often tightly coupled.
  • Poor-network behavior is commonly an afterthought.

Blossom Fire architecture

Control plane (Nostr)

  • Identity & signatures
  • Authorization flow
  • Viewer policy messages

Data plane (Blobs)

  • Chunk payload storage
  • Retryable fetch model
  • Transport decoupled from identity

Protocol flow

  • Viewer → Admin: access_req
  • Broadcaster announces chunk metadata on Nostr
  • Broadcaster → Admin: adminkey
  • Admin → Viewer: viewerkey
  • Viewer fetches blob, verifies authorization, plays

Sovereignty properties

Operator control

Self-host relay, blob service, and front-end. No mandatory platform dependency.

Portable identity

Nostr keys and events survive client/platform churn.

Explicit policy

Access is message-driven and auditable, not hidden in a proprietary backend.

Composable stack

Swap transport/storage without rewriting identity semantics.

Why good on unreliable networks

  • Chunk-based delivery handles jitter and intermittent connectivity.
  • Retry behavior tolerates transient missing chunks.
  • Out-of-order metadata can still converge to playable state.
  • Fallback modes preserve service when strict pipelines fail.

Pragmatic browser strategy

  • MSE path for richer buffering and smoother continuous playback.
  • Compatibility chunk mode for browsers/conditions where MSE is brittle.
  • Auto-recovery and explicit mode control to avoid black-screen failures.

Design principle: degrade gracefully, never silently die.

Use cases

  • Community livestreaming with user-owned infrastructure
  • Censorship-resistant event broadcast
  • Field/off-grid streaming where links are unstable
  • Research platform for sovereign media protocols

Roadmap

  • Stabilize Chrome MSE path (fMP4/CMAF exploration)
  • Adaptive bitrate/chunk policy for volatility
  • Deeper observability: mode, queue depth, drift, failure reason
  • Hardened key lifecycle and operator controls

Takeaway

Blossom Fire is a practical proof that live video can be:

  • decentralized in architecture
  • sovereign in operation
  • resilient under unreliable networks
